Name: explain-using-python

Purpose
-------
Offer analogies to Python ecosystems (Flask/Django) for backend patterns,
and plain-language explanations when Python is not a direct comparable.

When to use
-----------
- Backend Node/Express code or architecture diagrams where a Python
  comparison is requested.

Behavior
--------
- Show simple mapping examples (Express route -> Flask route), explain
  ORM parallels, and call out where concerns don't map cleanly.
